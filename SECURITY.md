# Security Policy for Fly High Local

Thank you for helping keep Fly High Local secure. We take the security of our services and our clients' digital storefronts seriously.

## Supported Versions

We actively provide security updates and patches for the current live deployment branch:

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public GitHub issue. Instead, report it directly to our development and founding team.

### How to Report
Please send an email to **flyhighlocal@gmail.com** with the following details:
* A description of the vulnerability and its potential impact.
* Detailed steps to reproduce the issue (including proof-of-concept scripts or screenshots if applicable).
* Any specific configurations or environment details required to trigger it.

### What to Expect
* **Acknowledgment:** We will acknowledge receipt of your report within 48 hours.
* **Triage:** Our software development head will review and validate the vulnerability.
* **Resolution:** We will keep you updated as we work toward fixing the issue and coordinate a public release timeline if a disclosure is warranted.

Thank you for your cooperation in protecting local neighborhood brands!
